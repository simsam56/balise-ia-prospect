# UI Audit - Balise-IA Prospect (2026-03-05)

## Contexte

Audit de l'interface pour un usage solo-entrepreneur (Balise-IA) en prospection B2B quotidienne:
- tri rapide des leads
- suivi statut + notes
- export vers outreach
- import CSV heterogene

## Constats initiaux

1. Hierarchie visuelle trop plate sur la page Leads.
2. Peu d'indicateurs de pilotage en haut de page.
3. Import CSV sans mapping explicite de colonnes.
4. Actions de prospection (copie email / LinkedIn) peu visibles.
5. Coherence visuelle globale perfectible (fond, contrastes, rythme de layout).

## Correctifs appliques

1. Refonte page Leads:
- hero avec actions principales (recalcul, export),
- cartes KPI (volume, priorite A, leads chauds, score moyen),
- tableau condense avec blocs Entreprise/Contact lisibles,
- actions rapides: copier email, ouvrir LinkedIn,
- pagination et taille de page ajustable.

2. Refonte import CSV:
- mapping colonne source -> champ canonique,
- mapping suggere automatique,
- preview mappee (10 lignes),
- import lance avec mapping actif.

3. Coherence UI globale:
- tokens visuels (fond, cartes, bordures),
- style systeme plus net sur inputs/selects/buttons,
- header harmonise.

4. Fiches:
- export CSV d'une entreprise depuis la vue entreprise,
- action "copier email" + "ecrire un email" sur fiche contact.

## Resultat attendu

- Gain de vitesse sur qualification et priorisation.
- Moins d'erreurs d'import CSV grace au mapping explicite.
- Parcours prospection plus direct pour email/LinkedIn/outreach.
